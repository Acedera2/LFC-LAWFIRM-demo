## 💻 Run a local MySQL quickly (recommended)

If you don't have MySQL running locally, you can start one via Docker Compose included in this repo:

1. From the repo root, start MySQL:

```powershell
docker-compose up -d
```

2. Update `server/.env` (the repo already points to root:root credentials for convenience) and run the safe prisma generator which attempts to remove tmp locks on Windows:

```powershell
cd server
npm run prisma:generate:safe
npm run prisma:migrate
npm run seed
```

3. Start the dev servers:

```powershell
npm run dev
```

This will start the backend at `http://localhost:5000` and the client on a vite port (usually `5173`).

## 🆘 QUICK TROUBLESHOOTING

Problem: "Can't connect to MySQL"
Solution: Run TEST_MYSQL.bat to find correct password

Problem: "Port already in use"
Solution: 
   netstat -ano | findstr :5000
   taskkill /PID <NUMBER> /F

Problem: "Module not found"
Solution:
   cd server && npm install
   cd ../client && npm install

Problem: "Database doesn't exist"
Solution:
   cd server
   mysql -u root -p
   CREATE DATABASE legal_field_consultancy;
   npx prisma migrate deploy
